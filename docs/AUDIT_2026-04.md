## 5. earnings_history Schema

### Schema (aus trades.db)

```sql
CREATE TABLE earnings_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,
    earnings_date DATE NOT NULL,
    fiscal_year INTEGER,
    fiscal_quarter TEXT,
    eps_actual REAL,
    eps_estimate REAL,
    eps_surprise REAL,
    eps_surprise_pct REAL,
    time_of_day TEXT,
    source TEXT DEFAULT 'marketdata',
    collected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(symbol, earnings_date)
);
CREATE INDEX idx_eh_symbol ON earnings_history(symbol);
CREATE INDEX idx_eh_date ON earnings_history(earnings_date);
CREATE INDEX idx_eh_symbol_date ON earnings_history(symbol, earnings_date);
```

### Beispieldaten (LIMIT 3)

```
id  | symbol | earnings_date | fiscal_year | fiscal_quarter | eps_actual | eps_estimate | eps_surprise | eps_surprise_pct | time_of_day   | source
68  | ABNB   | 2021-05-14    | 2021        | Q1             | -1.95      | -1.16        | -0.79        | -68.1            | after close   | yfinance
70  | ABNB   | 2020-11-16    | 2020        | Q3             | NULL       | NULL         | NULL         | NULL             | after close   | marketdata
71  | ABNB   | 2020-09-02    | 2020        | Q2             | NULL       | NULL         | NULL         | NULL             | during market | marketdata
```

### Antwort zur Frage

**Ja.** Die Tabelle enthaelt `eps_estimate REAL` als separate Spalte. Die Spalte ist nullable. In aelteren Zeilen (source = 'marketdata') sind eps_estimate und eps_actual haeufig NULL. In neueren Zeilen (source = 'yfinance') sind Estimate und Actual mit Werten befuellt.

Zusatzinfo: Das Schema enthaelt eine Spalte `source` (DEFAULT 'marketdata') und `collected_at` (TIMESTAMP), die in CLAUDE.md nicht dokumentiert sind. Der CLAUDE.md-Kommentar `source TEXT DEFAULT 'yfinance'` stimmt nicht mit der tatsaechlichen DB-Definition (`DEFAULT 'marketdata'`) ueberein.
