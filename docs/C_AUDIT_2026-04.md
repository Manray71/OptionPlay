## 3. Integration mit bestehendem Code

### 3a. MultiStrategyScanner — Scan-API

```python
# src/scanner/multi_strategy_scanner.py:1269
async def scan_async(
    self,
    symbols: list[str],
    data_fetcher: Callable,
    ...
) -> ScanResult
```

Der Scanner nimmt `data_fetcher` als Callable entgegen — nicht standalone aufrufbar.  
Der korrekte Pfad ist über `DailyRecommendationEngine.get_daily_picks()` (die den Scanner
intern orchestriert und den richtigen `data_fetcher` injiziert):

```python
# src/services/recommendation_engine.py:365
async def get_daily_picks(
    self,
    symbols: list[str],
    data_fetcher: Callable[..., Any],
    max_picks: int = 3,
    vix: Optional[float] = None,
    options_fetcher: Optional[Callable[..., Any]] = None,
) -> DailyRecommendationResult
```

**Konsequenz:** Telegram-Scheduler ruft `DailyRecommendationEngine.get_daily_picks()`,
nicht direkt `MultiStrategyScanner.scan_async()`. Dem Beispiel der Handler folgen
(`src/handlers/picks_composed.py`).

### 4c. Datenfluss

```
APScheduler (3× täglich)
  └─→ DailyRecommendationEngine.get_daily_picks(symbols, data_fetcher)
        └─→ DailyRecommendationResult (list[DailyPick])
              └─→ notifier.format_pick(pick)
                    └─→ Telegram-Message + InlineKeyboard
                          └─→ bot.send_message(chat_id, ...)

User drückt Button:
  ✅ DEAL ÖFFNEN
    └─→ callback_handler
          └─→ ShadowTracker.log_trade(source="telegram_bot", ...)
                └─→ trade_id → Bestätigungsnachricht
  ⏭ SKIPPEN
    └─→ callback_handler → Message löschen oder editieren (kein Trade)
  ⏰ SPÄTER
    └─→ callback_handler → APScheduler.add_job(remind, run_once, +Xh)
```

### 4d. Scheduler-Entscheidung

**APScheduler im Bot-Prozess** (empfohlen) statt zweitem Cron:

```python
# scheduler.py
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler(timezone="America/New_York")
scheduler.add_job(run_scan, "cron", hour=15, minute=30)  # Morning
scheduler.add_job(run_scan, "cron", hour=13, minute=0)   # Midday
scheduler.add_job(run_scan, "cron", hour=15, minute=45)  # Evening
scheduler.start()
```

Vorteile: Ein Prozess, kein separater Cron-Eintrag, Fehler im Bot-Log sichtbar.  
Nachteil: Bot muss laufen — macOS-Schlafen/Neustart bricht Jobs.  
→ Falls Robustheit kritisch: LaunchAgent als Wrapper um `run_telegram_bot.sh`.
